The React port keeps open/booked state as two `Set`s and lets conditional rendering do what `classList.toggle` + manual `innerHTML` rebuilding used to — verified against the same scenarios that already test the original (multiple rows can be open at once, `aria-expanded` stays in sync, booking updates the button text and disables it without a page reload):

```js
const [openIds, setOpenIds] = useState(new Set());
const [bookedIds, setBookedIds] = useState(new Set());

const toggleClass = (id) => {
  setOpenIds((prev) => {
    const next = new Set(prev);
    next.has(id) ? next.delete(id) : next.add(id);
    return next;
  });
};
```

**Production note:** the accordion markup above is real Bootstrap (`accordion`, `accordion-button`, `accordion-collapse`) — on afjcardiff.com, the same HTML pairs with Bootstrap's own `Collapse` API instead of React state:

```js
// Vanilla JS: bootstrap.Collapse instead of a React Set
document.querySelectorAll('.accordion-button').forEach((btn) => {
  const target = document.querySelector(btn.dataset.bsTarget);
  new bootstrap.Collapse(target, { toggle: false }); // multiple can stay open — no data-bs-parent
});
```

### How it got here — static markup, then a working port

The original is vanilla JS manipulating the DOM directly — global mutable variables, `classList.add`/`remove` to show/hide views, `innerHTML` string-building for the cart list:

```js
// original: static-file version, direct DOM manipulation
let cart = [];
let currentSize = 'M', currentQty = 1;

function addToCart() {
    cart.push({ name: activeProduct.name, price: activeProduct.price, size: currentSize, qty: currentQty });
    updateCartUI();
    showToast('Added to basket');
    openCart(); // side effect: also opens the drawer
}

function updateCartUI() {
    document.getElementById('cartCount').textContent = cart.length;
    const itemsEl = document.getElementById('drawerItems');
    itemsEl.innerHTML = cart.map(item => `<div class="cart-line">...</div>`).join('');
}
```

The React port replaces every one of those patterns with the equivalent state-driven idiom — same behaviour, verified against the same 14 Gherkin scenarios that already test the original (quantity floor of 1 with no ceiling, size selection reflected via `aria-pressed`, adding an item auto-opens the cart, removing the only item empties it without erroring):

```js
// React port: state drives the render, not the other way round
const [cart, setCart] = useState([]);
const [size, setSize] = useState('M');
const [qty, setQty] = useState(1);

const addToCart = () => {
  setCart((prev) => [...prev, { ...activeProduct, size, qty }]);
  showToast('Added to basket');
  setCartOpen(true); // same side effect, now just a state update
};
```

The cart list itself goes from an `innerHTML` template-string loop to a real `.map()` returning JSX — no string concatenation, no risk of a stray unescaped character breaking the markup.

**Production note:** Docusaurus needs React to render anything, but afjcardiff.com doesn't use React — it's plain Bootstrap 5 + vanilla JS, confirmed by the actual "Coming Soon" page currently live. So the component above uses real Bootstrap classes (`btn`, `card`, `offcanvas`, `list-group`), but the *code that would actually paste into afjcardiff.com* is the vanilla JS version, not the JSX:

```html
<!-- The Bootstrap markup is identical either way — only the interactivity differs -->
<div class="offcanvas offcanvas-end" id="cartDrawer" tabindex="-1">
  <div class="offcanvas-header">
    <h5 class="offcanvas-title">Your basket</h5>
    <button type="button" class="btn-close" data-bs-dismiss="offcanvas"></button>
  </div>
  <div class="offcanvas-body" id="drawerItems"></div>
</div>
```

```js
// Vanilla JS: Bootstrap's own Offcanvas API instead of React state
const drawer = new bootstrap.Offcanvas('#cartDrawer');
function addToCart() {
  cart.push({ name: activeProduct.name, price: activeProduct.price, size: currentSize, qty: currentQty });
  document.getElementById('cartCount').textContent = cart.length;
  document.getElementById('drawerItems').innerHTML = cart.map(item =>
    `<div class="list-group-item">${item.name} — £${item.price}</div>`
  ).join('');
  drawer.show(); // Bootstrap's real JS API, not classList.toggle
}
```

The fifth possible journey, the staging password gate, stays out of scope here since it's an internal dev tool a customer never sees, not a shopping flow.
