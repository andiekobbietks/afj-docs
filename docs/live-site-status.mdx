# How This Fits the Live Site

This library is a reference, not the live site itself. AFJ Cardiff's actual production site — custom CMS, merch shop with Stripe, and a planned subscription courses area — is being built separately. This page is the honest status of that build and how the two connect, current as of the last review.

## Current status

The full build is continuing rather than being replaced with a rushed placeholder. That decision was made deliberately after external feedback flagged the absence of a live site as a credibility gap with prospective partners. A fast landing-page alternative was considered and kept on the table as a bridge, not adopted as a substitute: the interim public presence is a tidied Linktree and a Canva one-pager, not a stand-in website.

**Trade-off, stated plainly:** this preserves brand quality control at the cost of a continued short-term credibility gap, and every week it slips, more of AFJ Cardiff's growing content sits undiscoverable behind an interim link page instead of a real site. That's a known, accepted cost, not an oversight.

## What this library is for, once the build resumes

This is the design-system reference the real build should implement against — tokens, components, both themes, and the commerce and booking flows already mocked up end to end. It exists so the actual site doesn't have to make brand and component decisions from scratch mid-build; they've already been made and tested here.

## Launch checklist this library doesn't cover

A handful of legal and compliance requirements attach the moment the real site goes live. None of them apply yet, but all of them belong on the go-live checklist rather than being discovered after launch:

- A cookie notice and website terms
- A privacy notice
- Accessibility consideration under the **Equality Act 2010**, given AFJ Cardiff is a public-facing arts organisation
- A UK-only site is still reachable by EU visitors by default, so the EU's cookie-consent standard applies in practice even though AFJ Cardiff itself only operates in the UK

None of this blocks using this component library now — it's simply not what this library documents.
