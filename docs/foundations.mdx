# Foundations

Design tokens, typography, and the icon system — the base layer every component builds on.

## Design tokens

Three themes now live side by side: the pixel-sampled Wine/Gold dark system, the Scrollytelling red/orange editorial system, and Day mode, the beige surfaces ( --afj-bg #F1EFE8 , --afj-beige #ECE7D8 , --afj-ink #1a1712 ). Toggle above, every preview updates at once.

  <TokenSwatch color="#141116" label="--bg-offset" />

  <TokenSwatch color="#C99552" label="--accent-1" />

**Preview**

```css
[data-theme="day"] {
  --bg-base: #F1EFE8;
  --bg-panel: #ECE7D8;
  --text: #1a1712;
  --accent-1: #C99552; --accent-2: #A43E53;

```

**What Bootstrap doesn't provide here, and how it's built anyway:** Bootstrap has no "color swatch" component — nothing in its component library is meant to display an arbitrary color value as a labeled block. The swatch grid above is still mostly real Bootstrap: `d-flex`, `flex-wrap`, and `gap-2` (all real utility classes) handle the layout entirely. The one thing Bootstrap genuinely can't do is the swatch's fill color itself, since that's dynamic per-token data, not a fixed set of Bootstrap's own theme colors — so that one property (`background`) stays inline. The rule of thumb used throughout this rebuild: reach for a real Bootstrap utility class first, and only fall back to inline styles or custom CSS for the specific property Bootstrap has no opinion on at all.

## Typography

Wine/Gold and Day mode stay sentence case in Montserrat. Scrollytelling goes uppercase in Space Grotesk with a gradient clip.

import GradientHeading from '@site/src/components/GradientHeading';

<GradientHeading as="p">Impact through dance</GradientHeading>

**Preview**

```html
<p class="demo-heading gradient">Impact through dance</p>
```

**What Bootstrap doesn't provide here, and how it's built anyway:** gradient-clipped text isn't a Bootstrap feature at all — no utility class, no component, no theming hook. Bootstrap's own `.text-primary` and similar classes only ever apply a flat `color`, never a `background` with `-webkit-background-clip: text`. This is genuinely outside what any component framework provides, Bootstrap or otherwise — it's a raw CSS technique (the same one works identically in a Tailwind or plain-CSS project), so the honest answer is: write the three lines of CSS that do it, once, and reuse the class everywhere the effect is needed, rather than pretend a utility class exists that doesn't:

```css
.gradient-heading {
  background: linear-gradient(135deg, var(--text) 30%, var(--accent-1) 70%, var(--accent-2) 100%);
  -webkit-background-clip: text;
  background-clip: text;
  -webkit-text-fill-color: transparent;
}
```

## Iconography

No custom icon set exists for AFJ Cardiff yet, so the working decision is to use Lucide (open-source, MIT-licensed, actively maintained, consistent 24×24 grid) rather than mixing icon sets ad hoc as new UI gets built. Every icon below is drawn with stroke="currentColor" , so it automatically matches whatever text colour the surrounding theme sets, no separate icon-colour token needed.

Rule of thumb: an icon never appears without a text label nearby, except inside a button whose action is already obvious from context (a lone × to dismiss a toast, a search glass inside a search field). Anything ambiguous, a settings gear, a generic arrow, gets a label. This avoids the single biggest accessibility failure mode of icon-only UI: a shape that means something different to every reader.

**Sizing scale**

```html
<svg width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">...</svg>
/* 32px icons drop stroke-width to 1.5 so the line weight doesn't look heavy at that size */
```
