## Pixel-sampled source

Every value below was read directly out of the live scrollytelling site's own source CSS ( gregarious-faun-060e2f.netlify.app ), not estimated off a screenshot, not eyeballed from a rendered page. This is the same discipline that caught the original brand mistake: the wine/gold gradient documented on this whole page only exists because pixel-sampling the master logo file surfaced the real #C99552 → #A43E53 values against a guessed CodeRabbit orange that had been wrongly applied. Applying that same rigour here means the Scrollytelling theme in this library is provably accurate to what's actually deployed, not a close approximation of it, which matters the moment someone builds a new page off this doc and expects the colours to match production exactly.

This is why "Pixel-sampled source" sits first in the Scrollytelling group, ahead of the components it powers: everything below, the hero, the story card, the manifesto, the signup box, is built from these five sampled values and two typefaces. Change the sample and every component downstream should be re-verified against it, the same way a wine-scale correction upstream would ripple through every button and badge in the Wine/Gold theme.
