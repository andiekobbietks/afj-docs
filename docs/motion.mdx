## Where Lottie fits

None of the components built so far use motion beyond CSS transitions and scroll-scrubbing. Lottie (lightweight vector JSON animation, renders via a single `<lottie-player>` web component, no video weight) is worth reserving for the handful of moments below where a static icon or CSS transition genuinely undersells what's happening, not sprinkled everywhere.

Being honest about what's not here yet: there's no pre-built Lottie animation on the marketplace that actually depicts Azonto or the specific step sequence documented in the scroll-scrubbed stage (The Signal, The Grounding, The Dialogue), that would need a custom commission from a LottieFiles community animator working from the same reference frames already used for the scroll-scrubbing images, not a stock download. Everything else above (cart, checkmark, loader) has genuine ready-made free options; the dance-specific motion does not.

**Integration snippet**

```html
<script src="https://unpkg.com/@lottiefiles/lottie-player@latest/dist/lottie-player.js"></script>

<lottie-player
src="https://assets.lottiefiles.com/packages/[chosen-animation].json"
background="transparent"
speed="1"
style="width:64px;height:64px"
autoplay>
</lottie-player>
/* loop only on ambient/idle animations (hero cue); never loop a
   success or confirmation animation, it should play once and stop,
   or it stops reading as confirmation and starts reading as decoration */
```
